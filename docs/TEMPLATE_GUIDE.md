# 評価表テンプレート差し替え手順

非エンジニアの方でも、次の手順で評価表を変更できます。
**進行中の考課期間には反映されません。** 次の夏季・冬季考課から適用されます。

## ファイルの場所

| ファイル | 用途 |
|----------|------|
| `backend/templates/self_evaluation_r8_summer.json` | 自己評価表 |
| `backend/templates/assessment_r8_summer.json` | 考課表（評価者用） |

将来の期間用には、例として `self_evaluation_r9_winter.json` のように **新しいファイル名** で追加します。

## 管理画面からの手順（v1 以降）

1. 人事・経営層アカウントでログイン
2. 「テンプレート管理」を開く
3. 「ファイルを選択」で新しい JSON をアップロード
4. **プレビュー** で項目数・点数段階を確認
5. エラーがなければ「次の考課期間から適用」をクリック

エラーがある場合は、日本語メッセージで不足項目が表示されます。

## よくある変更パターン

### 項目の文言を変える

`items` 配列の中の `element`（評価要素）や `criteria`（評価基準）の `text` を編集します。

```json
{
  "id": "item_01",
  "label": "①",
  "name": "自己管理",
  "element": "ここに評価要素の文章",
  "criteria": [
    { "score": 10, "text": "10点の基準文" },
    { "score": 8, "text": "8点の基準文" }
  ]
}
```

### 点数段階を変える

各 `criteria` の `score` を変更します。自己評価表と考課表で **同じ点数** に揃えてください。

### 項目を増やす・減らす

`items` 配列にオブジェクトを追加・削除します。
`id` は重複しないようにしてください（例: `item_11`）。

### 自由記述欄を変える

`text_fields` 配列の `label` や `instruction` を編集します。

## 注意事項

- JSON は **カンマの付け忘れ・余分なカンマ** に注意（編集後は必ず管理画面のプレビューで確認）
- 自己評価表（`type: self_evaluation`）と考課表（`type: assessment`）は **別ファイル** です
- 考課表の `items` は自己評価表と **同じ id** に揃えると、参照表示が正しく動きます
- 変更前のファイルは必ずバックアップを取ってください

## 困ったとき

- 管理画面のエラーメッセージをそのままメモして開発担当に連絡
- 元のファイル（`self_evaluation_r8_summer.json` など）に戻せば復旧できます
